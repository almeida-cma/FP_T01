Sequência Decisão Repetição

SEQUÊNCIA

passo a passo | ordem importa

🍳 Receita: Bolo de Chocolate
Fazer um bolo exige seguir os passos na ordem certa. Se misturar os ingredientes antes de pré-aquecer o forno, pode dar errado!
1. Pré-aqueça o forno a 180°C 2. Misture farinha, açúcar e ovos 3. Adicione chocolate em pó 4. Leve ao forno por 30 min 5. Espere esfriar e sirva
🧠 Lógica de programação: Na programação, a sequência significa executar instruções uma após a outra, na ordem em que foram escritas. Assim como uma receita, se um passo for trocado, o resultado pode ser diferente! É a base de todo algoritmo.
⚽ Cobrança de Pênalti
Um jogador segue uma sequência: posicionar a bola, respirar, correr, chutar. Qualquer alteração pode comprometer o gol.
1. Posicione a bola na marca 2. Dê 3 passos para trás 3. Aguarde o apito 4. Corra em direção à bola 5. Chute com força no canto
⚽ Programação é igual: Um algoritmo também segue uma sequência lógica. Primeiro a entrada (posicionar), depois o processamento (correr, chutar) e por fim a saída (gol ou não). Ordem é tudo!
✈️ Preparar uma Viagem
Viajar exige passos em sequência: escolher destino, comprar passagem, fazer mala, ir ao aeroporto.
1. Escolha o destino 2. Pesquise passagens 3. Compre a passagem 4. Faça as malas 5. Vá ao aeroporto
🗺️ No código: Um programa também executa em sequência. Se você tentar "fazer as malas" antes de "escolher o destino", o programa não faria sentido. A sequência organiza o raciocínio.

DECISÃO

se... senão | escolhas | condições

💄 Rotina de Skincare
Escolher o produto certo depende do tipo de pele. SE é oleosa, use gel; SENÃO, use creme hidratante.
SE (tipo_pele == "oleosa") { usar("gel de limpeza") } SENAO { usar("creme hidratante") }
🧴 Decisão na programação: O "SE" testa uma condição. Se for verdadeira, executa um bloco; senão, executa outro. É como um "se isso, faça aquilo" do dia a dia.
🎮 Jogo: Personagem morreu?
Em um jogo, SE a vida chegar a zero, o personagem morre e o jogo acaba. SENÃO, ele continua.
SE (vida <= 0) { mostrar("Game Over") finalizar_jogo() } SENAO { continuar_jogo() }
🎮 Decisão nos games: Toda ação em jogos depende de decisões. Se o jogador apertar "pular" e houver obstáculo, ele desvia. São estruturas de decisão que criam interatividade.
🏀 Arremesso no Basquete
O técnico decide: SE o jogador está livre, arremesse; SENÃO, passe a bola.
SE (marcador_esta_perto == falso) { arremessar() } SENAO { passar_bola() }
🏀 Decisão no esporte: Assim como no código, os atletas tomam decisões baseadas em condições. "Se estiver marcado, passe; se não, finalize". É pura lógica!
☕ Café com ou sem açúcar?
Ao servir café, você decide: SE a pessoa pediu com açúcar, adicione; SENÃO, sirva puro.
SE (pediu_acucar == verdadeiro) { adicionar_acucar() } servir_cafe()
☕ Decisão simples: Um "SE" pode ser usado sozinho também. Se a condição for verdadeira, faz algo; se não, apenas segue em frente. É como perguntar antes de agir.

REPETIÇÃO

laços | enquanto | para

🏋️ Treino: 10 repetições
Um atleta repete o mesmo movimento várias vezes. "Enquanto não completar 10, continue".
repeticoes = 0 ENQUANTO (repeticoes < 10) { fazer_agachamento() repeticoes = repeticoes + 1 }
🏋️ Repetição nos treinos: O laço "ENQUANTO" repete um bloco enquanto uma condição for verdadeira. No código, é usado para repetir tarefas sem precisar escrever o mesmo comando várias vezes.
🎲 Jogo: Rodada de Dados
Enquanto ninguém vencer, o jogo continua. Cada rodada repete as jogadas.
ENQUANTO (vencedor == "") { jogador1.jogar_dado() jogador2.jogar_dado() vencedor = verificar_vencedor() }
🎮 Loop infinito? Não! Enquanto uma condição for verdadeira, o código repete. Quando a condição muda, o loop acaba. É assim que jogos mantêm a partida ativa até alguém ganhar.
💅 Aplicar esmalte nas unhas
Repetir o processo para cada unha: passar base, esmalte, secar. Até terminar todas.
PARA (unha = 1; unha <= 10; unha++) { aplicar_base(unha) aplicar_esmalte(unha) secar() }
💅 Laço "PARA": Usamos "PARA" quando sabemos exatamente quantas vezes repetir. Aqui, 10 vezes, uma para cada unha. É prático e evita repetir código manualmente.
🍞 Fatiar pão
Repetir o corte até que o pão esteja completamente fatiado.
ENQUANTO (pao_nao_fatiado) { fatiar() pedacos_fatiados++ }
🍞 Enquanto não terminar: O loop "ENQUANTO" é ótimo quando não sabemos exatamente quantas vezes repetir. Repete até que a condição seja satisfeita.
